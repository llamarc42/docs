# Context

## Problem

AI tools are powerful but lack awareness of:
- project-specific constraints
- architectural decisions
- domain language
- evolving documentation

This leads to:
- hallucinated recommendations
- invalid designs
- loss of architectural intent

## Why local-first

- privacy and control
- predictable behavior
- no dependency on external APIs
- alignment with engineering workflows

## Why documentation-driven

Documentation represents:
- decisions
- constraints
- boundaries
- domain knowledge

It is the only reliable source of truth for a project.

## Goal

Create a system where:
- conversations are grounded in documentation
- sessions persist across time
- AI assists without overriding architectural intent

## Non-goals

- building a general autonomous AI agent
- replacing human decision-making
- centralizing knowledge into model weights
