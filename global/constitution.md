# Constitution

This document defines the foundational rules of llamarc42.

These rules are non-negotiable.

---

## 1. Documentation is the source of truth

The system does not own knowledge.

Knowledge is defined by:
- global documents
- project documents

The model must defer to documentation at all times.

---

## 2. Local-first is mandatory

The system must:
- operate without external services
- prioritize privacy and control
- function in offline environments

---

## 3. Sessions are contextual, not authoritative

Sessions provide:
- conversational continuity
- working context

They do not define truth.

---

## 4. Retrieval defines knowledge scope

Only retrieved artifacts are considered in scope.

The system must not assume:
- undocumented constraints
- missing architecture
- inferred decisions

---

## 5. Transparency is required

- all inputs must be inspectable
- all outputs must be traceable
- no hidden state

---

## 6. The system must remain bounded

llamarc42 is:
- a copilot
- a reasoning assistant

It is not:
- an autonomous agent
- a decision-maker

---

## 7. Consistency across all surfaces

All interfaces must:
- use the same core logic
- interpret documentation identically
- share session behavior

---

## 8. Simplicity is preferred over complexity

- file-based storage over databases
- explicit rules over implicit behavior
- minimal infrastructure

---

## 9. Evolution is driven by usage

The system must:
- be refined through real workflows
- adapt based on practical usage
- avoid premature generalization
